## 2. 用例图（核心场景）

### 2.1 开发者日常流程

```
[开发者] --> (登录平台)
          --> (打开项目 IDE)
          --> (AI 代码补全)
          --> (Agent 自动重构)
          --> (Git Commit & Push)
          --> (CI/CD 自动部署)
          --> (监控面板查看)
```

### 2.2 AI 应用构建流程

```
[AI 开发者] --> (创建 Agent 项目)
            --> (配置工具和 MCP)
            --> (上传知识库文档)
            --> (可视化编排工作流)
            --> (沙箱测试运行)
            --> (发布到市场)
            --> (部署为 API 服务)
```

### 2.3 团队协作流程

```
[团队管理员] --> (创建组织)
             --> (邀请成员)
             --> (配置 RBAC)
             --> (设置预算)
[团队成员]   --> (加入组织)
             --> (共享知识库)
             --> (协作开发)
             --> (Code Review)
```

## 3. 功能需求优先级矩阵

### P0 — MVP 必须（月 1-3）

| 模块 | 功能 |
|------|------|
| 用户中心 | 注册/登录、OAuth、API Key |
| AI Chat | 多模型对话、流式输出、多会话 |
| Agent | 基础 Agent 执行、Tool Calling、沙箱 |
| RAG | 文档上传、Chunk、Embedding、搜索 |
| Code | Monaco 编辑器、AI 补全、终端、Git |
| MCP | 内置 Server（Filesystem/GitHub/SQL） |

### P1 — 核心增强（月 4-6）

| 模块 | 功能 |
|------|------|
| 用户中心 | RBAC、组织管理、用量统计 |
| AI Chat | Prompt 库、多模态、历史搜索 |
| Agent | 可视化编排、Agent 模板、执行监控 |
| RAG | GitHub 导入、Hybrid Search、Rerank |
| Code | Code Review、Diff 视图、主题定制 |
| Workflow | 拖拽编辑器、AI/HTTP/代码节点 |
| MCP | 自定义 Server、工具测试 |
| Deploy | Docker 部署、K8s 部署、域名 SSL |
| Monitoring | Dashboard、日志搜索 |
| Billing | 用量统计、预算告警、支付集成 |
| Admin | 用户管理、模型管理、审计日志 |

### P2 — 生态扩展（月 7-12）

| 模块 | 功能 |
|------|------|
| Agent | 多 Agent 协作、插件市场 |
| RAG | OCR、代码索引、高级 Rerank |
| Workflow | SQL/Email/Slack 节点、定时触发 |
| MCP | Notion/Jira/Figma Server |
| Deploy | 多云部署、Terraform、成本优化 |
| Monitoring | 告警规则、链路追踪、AI 异常检测 |
| Billing | 多支付方式、发票、团队配额 |
| Admin | 系统配置、数据导出 |

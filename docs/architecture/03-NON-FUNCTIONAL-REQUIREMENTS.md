# OmniDev AI Platform — 非功能性需求

## 1. 性能要求

### 1.1 响应时间

| 场景 | P50 | P95 | P99 | 测量方式 |
|------|-----|-----|-----|----------|
| 页面首屏加载 (SSR) | < 800ms | < 1.5s | < 3s | Lighthouse |
| API 请求（非 AI） | < 100ms | < 300ms | < 1s | 服务端 APM |
| AI 流式首 Token | < 500ms | < 1.5s | < 3s | SSE 首字节时间 |
| AI 流式 Token 速率 | > 30 tok/s | > 20 tok/s | > 10 tok/s | 客户端计时 |
| 文件上传 (10MB) | < 5s | < 10s | < 30s | 客户端计时 |
| RAG 检索 | < 200ms | < 500ms | < 1s | 服务端 APM |
| 代码编辑器按键延迟 | < 50ms | < 100ms | < 200ms | 前端性能监控 |
| 终端输入回显 | < 30ms | < 80ms | < 150ms | WebSocket RTT |
| 工作流节点执行 | 取决于节点类型 | - | - | Temporal Metrics |
| 沙箱冷启动 | < 3s | < 5s | < 10s | 容器启动时间 |
| 沙箱热启动 | < 500ms | < 1s | < 2s | 容器复用 |

## 2. 可用性要求

### 2.1 SLA 目标

| 服务层级 | SLA | 每月最大宕机 | 适用范围 |
|----------|-----|-------------|----------|
| Tier 1 (核心) | 99.95% | 21.9 分钟 | Auth, Chat API, IDE |
| Tier 2 (重要) | 99.9% | 43.8 分钟 | Agent, RAG, Deploy |
| Tier 3 (辅助) | 99.5% | 3.6 小时 | Admin, Monitoring, Billing |

### 2.2 故障恢复

| 指标 | 目标 | 说明 |
|------|------|------|
| RPO (恢复点目标) | < 1 分钟 | 数据库 WAL 流复制 |
| RTO (恢复时间目标) | < 5 分钟 | 自动故障转移 |
| 备份频率 | 每 6 小时 | 全量 + 持续 WAL |
| 备份保留 | 30 天 | 跨区域存储 |
| 灾难恢复 | < 4 小时 | 跨可用区/区域 |

### 2.3 降级策略

| 场景 | 降级行为 |
|------|----------|
| AI 模型不可用 | 自动切换备用模型，提示用户 |
| RAG 服务不可用 | 跳过检索增强，直接对话 |
| 沙箱不可用 | 排队等待，提示用户 |
| 存储服务不可用 | 返回缓存数据，限制写入 |
| 数据库主库宕机 | 自动切换到只读副本 |

## 3. 安全要求

### 3.1 认证与授权

| 要求 | 实现 |
|------|------|
| 身份认证 | JWT (RS256) + Refresh Token |
| Token 有效期 | Access: 15min, Refresh: 7d |
| 密码策略 | 最少 8 位，含大小写+数字+特殊字符 |
| OAuth 2.0 | GitHub, Google, GitLab |
| API Key | Bearer Token，支持范围限制 |
| RBAC | 角色: Owner, Admin, Member, Viewer |
| 资源级权限 | 项目/知识库/工作流独立权限 |

### 3.2 数据安全

| 要求 | 实现 |
|------|------|
| 传输加密 | TLS 1.3 (全链路) |
| 存储加密 | AES-256 (敏感字段) |
| 密码哈希 | bcrypt (cost=12) |
| API Key 哈希 | SHA-256 (存储哈希，不存明文) |
| PII 数据 | 加密存储，日志脱敏 |
| 数据隔离 | 租户级逻辑隔离（Schema/RLS） |
| SQL 注入 | 参数化查询，ORM 防护 |
| XSS | CSP 头，输入净化 |
| CSRF | SameSite Cookie + Token |

### 3.3 沙箱安全

| 要求 | 实现 |
|------|------|
| 容器隔离 | gVisor / Kata Containers |
| 资源限制 | cgroups v2 (CPU/Memory/Disk) |
| 网络隔离 | 默认无网络，白名单放行 |
| 文件系统 | 只读根 + tmpfs 工作区 |
| 逃逸防护 | Seccomp + AppArmor |
| 镜像安全 | Trivy 扫描，签名验证 |
| 运行时监控 | Falco 异常检测 |

### 3.4 审计要求

| 事件 | 记录内容 | 保留期 |
|------|----------|--------|
| 登录/登出 | 用户、IP、时间、设备 | 1 年 |
| API Key 操作 | 创建/使用/吊销 | 1 年 |
| 权限变更 | 角色、资源、操作者 | 2 年 |
| 数据访问 | 敏感数据读取记录 | 1 年 |
| 管理操作 | 所有 Admin 操作 | 3 年 |
| AI 对话 | 消息内容（可配置） | 按计划 |

## 5. 可观测性要求

### 5.1 指标 (Metrics)

**RED 方法 (Request-driven)：**
| 指标 | 说明 | 采集方式 |
|------|------|----------|
| Rate | 每秒请求数 | Prometheus Counter |
| Error | 错误率 | Prometheus Counter |
| Duration | 请求延迟分布 | Prometheus Histogram |

**USE 方法 (Resource-driven)：**
| 指标 | 说明 | 采集方式 |
|------|------|----------|
| Utilization | CPU/内存/磁盘使用率 | Node Exporter |
| Saturation | 队列深度、连接池使用 | Custom Exporter |
| Errors | 资源错误（OOM、磁盘满） | Kernel Events |

**业务指标：**
| 指标 | 说明 |
|------|------|
| AI Token 消耗 | 按模型/用户/功能 |
| 活跃用户数 | DAU/WAU/MAU |
| 对话数 | 按模型/功能 |
| Agent 执行数 | 成功率/失败率 |
| RAG 查询数 | 命中率/延迟 |
| 部署数 | 成功/失败/回滚 |

### 5.2 日志 (Logging)

**结构化日志格式 (JSON)：**
```json
{
  "timestamp": "2026-01-15T10:30:00Z",
  "level": "info",
  "service": "chat-engine",
  "trace_id": "abc123",
  "span_id": "def456",
  "user_id": "user_123",
  "message": "Chat completion started",
  "attributes": {
    "model": "gpt-4o",
    "token_count": 150
  }
}
```

**日志级别使用规范：**
| 级别 | 用途 |
|------|------|
| ERROR | 需要立即关注的错误 |
| WARN | 异常但可恢复的情况 |
| INFO | 关键业务事件 |
| DEBUG | 开发调试信息（生产关闭） |

### 5.3 链路追踪 (Tracing)

| 要求 | 实现 |
|------|------|
| 协议 | OpenTelemetry |
| 采样率 | 生产 10%，错误 100% |
| 传播 | W3C TraceContext |
| Span 粒度 | 每个服务调用/DB 查询/AI 调用 |
| 关联 | TraceID 贯穿日志/指标/追踪 |

### 5.4 告警规则

| 告警 | 条件 | 级别 | 通知渠道 |
|------|------|------|----------|
| 服务宕机 | 健康检查失败 > 1min | Critical | PagerDuty + Slack |
| 高错误率 | 5xx > 5% 持续 5min | Critical | Slack + Email |
| 高延迟 | P95 > 2s 持续 5min | Warning | Slack |
| CPU 过高 | > 80% 持续 10min | Warning | Slack |
| 内存过高 | > 85% 持续 5min | Warning | Slack |
| 磁盘空间 | < 20% 剩余 | Warning | Slack |
| 消息积压 | Kafka Lag > 10K | Warning | Slack |
| 沙箱队列 | 等待 > 5min | Warning | Slack |
| AI 模型故障 | 连续 3 次失败 | Critical | Slack + Email |
